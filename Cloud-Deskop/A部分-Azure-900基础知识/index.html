# 问题 1

  1. 平台即服务（PaaS）解决方案 Azure 中的 Web 应用提供了对承载应用程序的操作系统。[x]
  2. 平台即服务（PaaS）解决方案 Azure 中的 Web 应用提供了扩展紫红打开平台。 [√]
  3. 平台即服务（PaaS）解决方案 Azure 中的 Web 应用提供专业的不断添加功能的开发服务定义应用程序。 [√]

# 解答

  1. 否
    原因：PaaS 解决方案不提供对操作系统的访问。 Azure Web Apps 服务提供供您托管 Web 应用程序的环境。 在幕后，Web 应用程序托管在运行 IIS 的虚拟机。 但是，您无法直接访问虚拟机，操作系统或 IIS。
  2. 是
    原因：在 Azure 中托管 Web 应用程序的 PaaS 解决方案确实提供了自动扩展平台的能力。这称为自动缩放。 在幕后，Web 应用程序托管在运行的虚拟机上信息系统。 自动缩放意味着添加更多负载平衡的虚拟机来托管 Web 应用程序。
  3. 是
    原因：PaaS 提供了一个框架，开发人员可以在此基础上开发或定制基于云的应用程序。 PaaS 开发工具可以减少使用预编码应用程序编写新应用程序所需的时间平台内置的组件，例如工作流、目录服务、安全功能、搜索等上。

链接：https://azure.microsoft.com/en-gb/overview/what-is-paas/

# 问题 2

  1. Azure 提供了不同资本之间的灵活性支出（资本支出）和运营 Exponditure（运营成本）。 [√]
  2. 如果你创建了两个 Azure 虚拟机使用 B2S 大小，每个虚拟机将总是产生相同的每月成本。 [x]
  3. 当 Azure 虚拟机停止时，你继续支付与虚拟机。[√]

# 解答

  1. 是
    原因：传统上，IT 支出被视为资本支出 (CapEx)。 今天，随着搬到云和现收现付模式，组织有能力扩展他们的预算，并且将其 IT 资本支出成本转为运营支出 (OpEx)。 这种灵活性，在会计方面，由于购买软件、云存储和其他 IT 相关的 “即服务” 模式，现在是一种选择资源。
  2. 否
    原因：使用相同大小的两个虚拟机可能具有不同的磁盘配置。 因此，每月成本可能不同。
  3. 是
    原因：当 Azure 虚拟机停止时，您无需为虚拟机付费。 但是，您仍然需要支付用于与虚拟机相关的存储成本。 最常见的存储成本是磁盘附加到虚拟机。 还有其他与虚拟机相关的存储成本例如用于诊断数据和虚拟机备份的存储。

链接：https://meritsolutions.com/capex-vs-opex-cloud-computing-blog/

# 问题 3

当你实现软件即服务（SaaS）时解决方案，你负责 (D)
A. 配置高可用性 [x]
B. 定义可伸缩规则 [x]
C. 安装 SaaS 解决方案 [x]
D. 配置 SaaS 解决方案 [√]

# 解答

原因：

  1. 当你实施软件即服务（SaaS）解决方案时，你有责任配置 SaaS 解决方案，其他一切由云提供商管理。
  2. SaaS 需要最少的管理，云提供商负责管理一切，而最终用户只是使用该软件。
  3. 软件即服务 (SaaS) 允许用户通过 Internet 连接和使用基于云的应用程序。
    常见示例是电子邮件、日历和办公工具（例如 Microsoft Office 365）。
  4. SaaS 提供了一个完整的软件解决方案，您可以从云中以现收现付的方式购买服务提供者。 您为您的组织租用应用程序，您的用户通过 Internet，通常使用 Web 浏览器。 所有底层基础设施、中间件、应用软件和应用数据位于服务提供商的数据中心。 服务提供商管理硬件和软件和适当的服务协议，将确保应用程序的可用性和安全性以及您的数据。

链接：https://azure.microsoft.com/en-in/overview/what-is-saas/
链接：https://docs.microsoft.com/en-gb/learn/modules/principles-cloud-computing/5-types-of-cloud-services

# 问题 4

你有一个包含多个服务器的本地网络。你将所有服务器迁移到 Azure。
如果单个 Azure 数据中心长时间脱机，你需要推荐一种解决方案以确保某些服务器可用。
你应该在建议中包含哪些内容（ A ）？
A. 容错 B. 弹性 C. 可扩展性 D. 低延迟

# 解答

原因：

  1. 容错是系统在其某些功能发生故障时继续运行的能力组件。
  2. 在这个问题中，您可以拥有跨数据中心复制的服务器。
  3. 可用区扩展了您必须保持应用程序可用性和虚拟机上的数据。可用区是 Azure 区域内唯一的物理位置。每个区是由一个或多个配备独立电源、冷却和网络的数据中心组成。至确保弹性，在所有启用的区域中至少有三个独立的区域。物理区域内的可用区分离可保护应用程序和数据免受数据中心故障的影响。借助可用区，Azure 提供业界最佳的 99.99% VM 正常运行时间 SLA。通过构建您的解决方案在区域中使用复制的 VM，您可以保护您的应用程序和数据免受数据中心丢失的影响。如果一个区域受到威胁，然后复制的应用程序和数据会立即在另一个区域中可用。

链接：https://docs.microsoft.com/en-us/azure/virtual-machines/windows/manage-availability

# 问题 5

托管其基础设施的组织 (C) 不再需要数据中心。
A. 在私有云中 [x]
B. 在混合云中 [x]
C. 在公共云中 [√]
D. 在 Hype-V 主机上 [x]

# 解答

原因：

  1. 私有云托管在您的数据中心。 因此，如果您使用的是私有云，则无法关闭数据中心。
  2. 公共云在外部托管，例如在 Microsoft Azure 中。 一个托管其的组织公共云中的基础设施可以关闭其数据中心。
  3. 公共云是最常见的部署模型。 在这种情况下，您没有要管理的本地硬件或保持最新 --- 一切都在您的云提供商的硬件上运行。
  4. Microsoft Azure 是公共云提供商的一个示例。
  5. 在私有云中，您在自己的数据中心创建云环境并提供自助访问为组织中的用户计算资源。 这为您的用户提供了公共云的模拟，但您仍对硬件和软件的购买和维护负全部责任您提供的服务。

链接：https://docs.microsoft.com/en-gb/learn/modules/principles-cloud-computing/4-cloud-deployment-models

# 问题 11

你的公司托管了一个名为 App1 的会计应用程序。该应用程序有公司所有客户使用。
App1 在每个月的前瞻周使用率较低。而在每个越的最后一周使用率很高。
Azure 云服务的哪些优势支持次磊使用模式的成本管理？（C）
A. 高可用性 [x]
B. 高延迟 [x]
C. 弹性 [√]
D. 负载均衡 [x]

# 解答

原因：

  1. 在这种情况下，弹性是能够在需要时提供额外的计算资源并减少不需要时计算资源以降低成本。 自动缩放是弹性的一个例子。
  2. 弹性计算是快速扩展或减少计算机处理、内存和存储的能力资源来满足不断变化的需求，而无需担心峰值的容量规划和工程用法。 通常由系统监控工具控制，弹性计算匹配资源量在不中断运营的情况下分配给实际需要的资源量。 有了云弹性，公司避免为未使用的容量或闲置资源付费，并且不必担心投资购买或维护额外的资源和设备。

# 问题 13

Azure 虚拟机和 Azure SQL 数据库使用哪种云部署解决方案？
Azure 虚拟机（A）
A. 基础设施即服务（IaaS） [√]
B. 平台即服务（PaaS） [x]
C. 软件即服务（SaaS） [x]
Azure SQL 数据库（B）
A. 基础设施即服务（IaaS） [x]
B. 平台即服务（PaaS） [√]
C. 软件即服务（SaaS） [x]

# 解答

原因：
Azure 虚拟机是基础结构即服务 (IaaS)。
基础设施即服务是最灵活的云服务类别。 它旨在为您提供完整的控制运行您的应用程序的硬件（IT 基础架构服务器和虚拟机 (VM)，存储、网络和操作系统）。 有了 IaaS，您无需购买硬件，而是租用它。
Azure SQL 数据库是平台即服务 (Paas)。
Azure SQL 数据库是一个完全托管的平台即服务 (PaaS) 数据库引擎，可处理大多数升级、打补丁、备份、监控等数据库管理功能，无需用户参与。 Azure SQL 数据库始终在 SQL Server 数据库的最新稳定版本上运行引擎和打补丁的操作系统具有 99.99% 的可用性。 Azure SQL 中内置的 PaaS 功能数据库使您能够专注于特定领域的数据库管理和优化活动，这对您的业务至关重要。

链接：https://docs.microsoft.com/en-gb/learn/modules/principles-cloud-computing/5-types-of-cloud-services
链接：https://docs.microsoft.com/en-us/azure/sql-database/sql-database-paas-index

# 问题 14

你有一个包含 100 台服务器的本地网络。
你需要推荐一种可为用户提供额外资源解决方案。该解决方案必须最大限度地减少资本和运营支出成本。
你应该在建议中包含哪些内容？（D）
A. 完全迁移到公有云 [x]
B. 额外的数据中心 [x]
C. 私有云 [x]
D. 混合云 [√]

# 解答

原因：
混合云是私有云和公共云的组合。
资本支出是用于基础设施（例如新服务器）的前期支出。
使用混合云，您可以继续使用本地服务器，同时在公共云（例如 Azure）。 在 Azure 中添加新服务器可将资本支出成本降至最低，因为您不会像在本地部署新服务器那样为新服务器付费。
不正确的答案：
A：将 100 台服务器完整迁移到公有云将涉及大量运营支出（迁移所有服务器的成本）。
B：额外的数据中心将涉及大量资本支出（新基础设施的成本）。
C：私有云托管在本地服务器上，这将涉及大量资本支出（托管私有云的新基础架构的成本）。

链接：https://docs.microsoft.com/en-gb/learn/modules/principles-cloud-computing/4-cloud-deployment-models

# 问题 15

  1. 要实现混合云模型。公司必须始终从私有云迁移模型。[x]
  2. 公司可以通过使用公共云来扩展起内部网络的容量。[√]
  3. 在公共云模型中，只有公司的来宾用户可以访问云。[x]

# 解答

原因：

  1. 否
    公司必须始终从私有云模型迁移以实施混合云是不正确的。
    您可以从公共云开始，然后将其与内部部署的基础设施相结合来实施混合云。
  2. 是的
    公司可以通过使用公共云来扩展其内部网络的容量。这是很常见的。
    当您需要更多容量而不是为新的本地基础架构付费时，您可以配置一个云环境，并使用 VPN 将您的本地网络连接到云环境。
  3. 否
    并非只有访客用户才能访问云资源。你可以给任何有账户的人 Azure Active Directory 访问云资源。有很多身份验证场景，但一个常见的一种是将本地 Active Directory 帐户复制到 Azure Active Directory 和提供对 Azure Active Directory 帐户的访问。另一种常用的身份验证方法是 “联合”，其中访问云资源的身份验证传递给另一个身份验证提供程序，例如本地 Active Directory。

链接：https://azure.microsoft.com/en-gb/overview/what-is-hybrid-cloud-computing/

# 问题 16

你计划将多个服务器从本地网络迁移到 Azure。
通过本地网络为服务器使用公共云服务有什么优势？（D）
A. 公共云归公司所有，而不是私人访问。[x]
B. 公共云是一种众包解决方案，可为企业提供增强云的能力。[x]
C. 所有公共云资源都可以被每个公众成员自由访问。[x]
D. 公共云是一个共享实体，多个公司各自使用云中的一部分资源。[√]

# 解答

原因：
公共云是一个共享实体，多个公司各自使用云中的一部分资源。 硬件资源（服务器、基础设施等）由云提供商管理。 多家公司在硬件资源上创建虚拟机和虚拟网络等资源。
不正确的答案：
A：公有云不为公众所有。 就 Microsoft Azure 而言，云由 Microsoft 拥有。
B：公共云不是众包解决方案。 就 Microsoft Azure 而言，云由 Microsoft 拥有。
C：公有云资源可以被每个公众成员自由访问是不正确的。 您为云订阅付费并为您的用户创建帐户以访问您的云资源。 在您创建用户帐户并提供适当的访问权限之前，没有人可以访问您的云资源。

你有 1000 台虚拟机托管正在数据中心的 Hyper-V 主机上。
你计划将所有虚拟机迁移到 Azure 即用即付订阅。你需要确定要用于计划的 Azure 解决方案的支出模型。你应该确定哪种支出模型？（A）
A. 操作 [√]
B. 松紧带 [x]
C. 首都 [x]
D. 可扩展的 [x]

# 解答

原因：
当你从本地迁移到公共云时，你将勉励的主要变化之一时从资本支出（购买硬件）转向运营支出（在使用时支付费用）。此转换还需要更仔细地管理你的成本。云的好处是，你只需要在不需要时关闭或者调整他的大小，就可以从根本上积极地影响你使用的服务的成本。

# 问题 21

将 Azure 云服务权益与正确的描述相匹配。
失败后仍然可用云服务（发生） ===> [容错]
故障后可以恢复的云服务（发生） ===> [灾难恢复]
在需要是快速之行的云服务（增加） ===> [动态可扩展性]
可以快速访问的云服务（从互联网上） ===> [低延迟]

# 解答

原因：

  1. 容错是服务的一个组件发生腹胀后保持可用的能力。例如：运行在多台服务器上的服务可以承受其中的一台服务器的故障。
  2. 灾难恢复时服务器发生付帐后的恢复。例如：在虚拟机出现付帐后从备份恢复虚拟机。
  3. 动态可扩展性时在服务堵在较重时将计算资源添加到服务的能力，例如：在虚拟机规模集中，当现有虚拟机负载过重时，会添加格外的虚拟机实例。
  4. 延迟时服务器响应请求的时间。例如：从 Web 服务器返回网页所需的时间，低延迟意味着地相应时间，这意味着更快的响应。

# 问题 22

  1. 要实现混合云模型，公司必须又内部网络。[x]
  2. 公司可以扩展其内部网络的计算资源使用混合云。 [√]
  3. 在公共云模型中，只有你公司的来宾用户可以访问云中的资源。[x]

# 解答

原因：

  1. 否
    公司必须失踪从内部网络迁移一实施混合云时不正确的。你可以从公共云开始，然后将其与本地基础架构想解和以实施混合云。
  2. 是
    企业可以通过佛念佛有云扩展其内部网络的计算资源，是很常见的。当你需要更多资源而不是为新的本地基础架构复位时，你可以配置云环境并使用 VPN 将本地网络链接到云环境。
  3. 否
    并非只有方可用户才能访问云资源。你可以授予拥有 Azure Active Directory 账户得任何人方云云资源得权限。有许多身份验证方案。但常见得一种时将本地 Active Directory 账户复制到 Azure Active Directory 并提供对 Azure Active Directory 账户访问。另一种常用得身份验证方法是 "联合", 其中访问云资源得身份验证传递给另一种身份验证提供程序，例如本地 Active Directory。

链接：https://azure.microsoft.com/en-gb/overview/what-is-hybrid-cloud-computing/

# 问题 24

你的公司有一个包含多个服务器的内部网络。该公司计划减少一下网络管理员的管理职责：
-- 备份应用数据
-- 替换失败的服务器硬件
-- 管理物理服务器的安全性
-- 更新服务器操作系统
-- 管理对共享文档的权限
该公司计划将几台服务器迁移到 Azure 虚拟机上，你需要确定在计划的迁移之后哪些管理职责将被取消，你应该确定哪两个责任？(AC) 每个正确答案提供了一个完整的解决方案。
A. 替换失败的服务器硬件 [√]
B. 备份应用程序数据 [x]
C. 管理物理服务器安全性 [√]
D. 更新服务器操作系统 [x]
E. 管理对共享文档的权限 [x]

# 解答

原因：
Azure 虚拟机运行在 Hyper-V 物理服务器上。物理服务器由微软拥有和管理，座位一个 azure 客户，你没有访问物理服务器的权限。微软负责更换失败的服务器硬件和物理服务器的安全，所以你不需要这么做。

错误答案：
B.Microsoft 无法控制你在虚拟机上运行的应用程序。因此，确保应用程序数据备份是你的责任。
D. 微软部管理你在虚拟机上运行的操作系统。因此，确保操作系统更新是你的责任。
E.Microsoft 无法控制你在虚拟机上托管的共享文件夹。因此，确保文件夹权限被正确配置是你的责任。
